NeuroSight Setup Instructions (Updated)

Prerequisites
- Python 3.11
- Node.js (18+ recommended)
- npm
- PostgreSQL + pgAdmin

1) Create Database
1. Open pgAdmin.
2. Create a database named brain_tumor.

2) Open Project
1. Clone the repository.
2. Open the project root in VS Code.

3) Backend Setup
1. Go to backend folder:
	cd backend
2. Create venv:
	py -3.11 -m venv .venv
3. Activate venv:
	.\.venv\Scripts\Activate
4. Upgrade pip tools:
	python -m pip install --upgrade pip setuptools wheel
5. Install backend requirements:
	pip install --no-cache-dir -r requirements.txt
6. Create environment file:
	copy .env.example .env
7. Edit .env and set real values:
	- DATABASE_URL: change the password to your own PostgreSQL password
	- SECRET_KEY: keep as-is (same for all team members)
	- Add SMTP settings from section 11 if you want email enrollment to work
8. Return to project root:
	cd ..

IMPORTANT — One-time warning for existing team members:
	backend/.env has been removed from git tracking (it should never have been committed).
	The next time you pull from stage-01, git will DELETE your local backend/.env.
	Before pulling: copy your backend/.env to a safe place (e.g. Desktop).
	After pulling: put it back. Future pulls will never touch it again.

4) Run Backend (Current Method)
1. From project root, run:
	npm run dev
2. API docs:
	http://127.0.0.1:8000/docs
3. Health endpoints:
	http://127.0.0.1:8000/health
	http://127.0.0.1:8000/health/model

Note:
- npm run dev at root uses backend/.venv Python directly.
- Uploaded MRI and document folders are served from:
  /uploaded_mris and /uploaded_docs

5) Frontend Setup and Run
1. Go to frontend folder:
	cd frontend
2. Install dependencies:
	npm install
3. Start frontend:
	npm run dev
4. Open URL shown by Vite (usually http://localhost:5173).

Optional: Run frontend from root
- npm run dev:frontend

6) Test Database Connection
1. Activate backend venv and run from backend folder:
	python test_db.py

If you use a separate test database URL:
- set TEST_DATABASE_URL=postgresql+psycopg2://<user>:<password>@<host>:5432/<dbname>
- python test_db.py

7) Create New User
1. Start backend.
2. Open API docs at http://127.0.0.1:8000/docs.
3. Use auth endpoint (register/signup) to create a new user.

 # Mobile Chatbot Instructions
8) Mobile Chatbot Initial Setup (run once before using the chatbot)
Note: The backend venv must be activated before running any of these steps.
      All commands run from the project root unless stated otherwise.

Step 1 - Install chatbot dependencies (inside activated backend venv):
	pip install -r backend/chatbot/requirements.txt

Step 2 - Download model.safetensors from Google Drive (NOT in git — too large):
	Download link:
		https://drive.google.com/file/d/1BGm0flcAK47gmsEV8-fEeVsZfPq9qUOk/view?usp=drive_link
	Place the downloaded file at:
		backend/chatbot/model/neurosight_distilbert/model.safetensors
	The following files ARE already in the repository (no download needed):
		backend/chatbot/model/faiss_index.bin
		backend/chatbot/model/qa_dataset_final.csv

	IMPORTANT: Without model.safetensors the chatbot microservice will not start.
	           The main backend (port 8000) will still work using the TF-IDF fallback.

Step 3 - Run the chatbot microservice in a separate terminal (port 8001):
	uvicorn backend.chatbot.microservice:app --port 8001

Step 4 - Run the main backend as usual in another terminal (port 8000):
	npm run dev
	   OR
	uvicorn backend.main:app --reload --port 8000

Both terminals must stay open for the chatbot to work.
If port 8001 is not running, the chatbot automatically falls back to
the TF-IDF classifier (Step 2) — no crash, but reduced answer quality.

10) Emergency feature (SOS alerts)
- Purpose: mobile SOS sends structured alerts to `/mobile/notify`. The backend persists an emergency `ChatMessage` and emits a realtime `emergency_alert` event to clinician dashboards via Socket.IO.
- Install Socket.IO client (frontend):
		cd frontend
		npm install --save socket.io-client
- Dev run order to test locally:
		1. Start backend: `npm run dev` (or `uvicorn backend.main:app --reload --port 8000`)
		2. Start frontend dashboard: `cd frontend && npm run dev`
		3. Start mobile app: `cd mobile && npm run dev`
		4. Trigger SOS from mobile; verify alerts appear on dashboard and realtime popup
- Notes:
	- `socket.io-client` is required by the dashboard to receive live emergency alerts (push). Without it the dashboard will only show alerts via polling endpoints.
	- Geolocation (`navigator.geolocation`) requires HTTPS on devices; `localhost` works for most dev setups.

--- Step B: Testing on a real phone ---
Each person uses their own local backend and database.
ngrok creates a public URL so your phone can reach your laptop's backend.

Your steps:
	1. Start your backend: npm run dev  (from project root)
	2. Open a second terminal and run ngrok:
	   .\ngrok.exe http 8000
	   (ngrok.exe is already in the project root)
	3. First time only — add your ngrok authtoken (sign up free at dashboard.ngrok.com):
	   .\ngrok.exe config add-authtoken YOUR_TOKEN
	4. Copy the URL ngrok shows (e.g. https://abc123.ngrok-free.app)
	5. Send the URL to Fathima Shameeha — she will update VITE_API_URL in Vercel and redeploy
	   (only she has access to the Vercel project)
	6. Once redeployed, open https://neurosight-app.vercel.app on your phone — login will work

Note: ngrok URL changes every time it restarts. Repeat steps 2–6 each session.
