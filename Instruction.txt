NeuroSight Setup Instructions (Updated)

Prerequisites
- Python 3.11
- Node.js (18+ recommended)
- npm
- PostgreSQL + pgAdmin

1) Create Database
1. Open pgAdmin.
2. Create a database named brain_tumor.

2) Open Project
1. Clone the repository.
2. Open the project root in VS Code.

3) Backend Setup
1. Go to backend folder:
	cd backend
2. Create venv:
	py -3.11 -m venv .venv
3. Activate venv:
	.\.venv\Scripts\Activate
4. Upgrade pip tools:
	python -m pip install --upgrade pip setuptools wheel
5. Install backend requirements:
	pip install --no-cache-dir -r requirements.txt
6. Create environment file:
	copy .env.example .env
7. Edit .env and set real values (especially DATABASE_URL and SECRET_KEY).
8. Return to project root:
	cd ..

4) Run Backend (Current Method)
1. From project root, run:
	npm run dev
2. API docs:
	http://127.0.0.1:8000/docs
3. Health endpoints:
	http://127.0.0.1:8000/health
	http://127.0.0.1:8000/health/model

Note:
- npm run dev at root uses backend/.venv Python directly.
- Uploaded MRI and document folders are served from:
  /uploaded_mris and /uploaded_docs

5) Frontend Setup and Run
1. Go to frontend folder:
	cd frontend
2. Install dependencies:
	npm install
3. Start frontend:
	npm run dev
4. Open URL shown by Vite (usually http://localhost:5173).

Optional: Run frontend from root
- npm run dev:frontend

6) Test Database Connection
1. Activate backend venv and run from backend folder:
	python test_db.py

If you use a separate test database URL:
- set TEST_DATABASE_URL=postgresql+psycopg2://<user>:<password>@<host>:5432/<dbname>
- python test_db.py

7) Create New User
1. Start backend.
2. Open API docs at http://127.0.0.1:8000/docs.
3. Use auth endpoint (register/signup) to create a new user.

 # Mobile Chatbot Instructions
8) Mobile Chatbot Initial Setup (run once before using the chatbot)
Note: The backend venv must be activated before running any of these steps.
      All commands run from the project root unless stated otherwise.

Step 1 - Install chatbot dependencies (inside activated backend venv):
	pip install -r backend/chatbot/requirements.txt

Step 2 - Download model.safetensors from Google Drive (NOT in git — too large):
	Download link:
		https://drive.google.com/file/d/1BGm0flcAK47gmsEV8-fEeVsZfPq9qUOk/view?usp=drive_link
	Place the downloaded file at:
		backend/chatbot/model/neurosight_distilbert/model.safetensors
	The following files ARE already in the repository (no download needed):
		backend/chatbot/model/faiss_index.bin
		backend/chatbot/model/qa_dataset_final.csv

	IMPORTANT: Without model.safetensors the chatbot microservice will not start.
	           The main backend (port 8000) will still work using the TF-IDF fallback.

Step 3 - Run the chatbot microservice in a separate terminal (port 8001):
	uvicorn backend.chatbot.microservice:app --port 8001

Step 4 - Run the main backend as usual in another terminal (port 8000):
	npm run dev
	   OR
	uvicorn backend.main:app --reload --port 8000

Both terminals must stay open for the chatbot to work.
If port 8001 is not running, the chatbot automatically falls back to
the TF-IDF classifier (Step 2) — no crash, but reduced answer quality.

9) Mobile App Setup and Run
1. Go to mobile folder:
	cd mobile
2. Install dependencies:
	npm install
3. Start mobile app:
	npm run dev
4. Open URL shown by Vite (usually http://localhost:5174).
